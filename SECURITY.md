# Security Policy

## Reporting a vulnerability

If you discover a security issue, **do not open a public GitHub issue first.**

Instead, before publication, go to the repository **Settings → Security → Private vulnerability reporting** and enable it. Once enabled, reporters can submit vulnerabilities via GitHub's private reporting flow. If the feature is not yet configured, do not publish this repository as accepting private security reports; instead, disclose responsibly through your own channels.

Include the following in any report:

- Description of the vulnerability and its impact.
- Steps to reproduce (pseudocode or minimal example is fine).
- Affected versions and suggested mitigation, if known.

We aim to acknowledge reports within 48 hours and will provide an estimated timeline for a fix. We follow a coordinated disclosure process and will credit reporters in the release notes with your permission.
